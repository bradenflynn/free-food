# Plan: 03-01-PLAN.md (Supabase Auth & RLS)

**Phase:** 3 (Security & Admin)
**Wave:** 1
**Autonomous**: No (Requires User to create an admin account)

## Goal
Secure the database so only authenticated users can edit event data.

## Requirements
- **SEC-01**: Supabase Auth enabled for admin login.
- **SEC-02**: Row Level Security (RLS) restricts "Approve/Reject" updates to authenticated users only.

## Tasks
```xml
<task id="1">
  <description>User enables 'Email/Password' auth in Supabase Dashboard (Authentication -> Providers).</description>
</task>
<task id="2">
  <description>User creates an admin account in Supabase Dashboard (Authentication -> Users -> Add User).</description>
</task>
<task id="3">
  <description>Update RLS policies to restrict UPDATE and DELETE to 'authenticated' role.</description>
</task>
```

## Verification Criteria
- Attempting to update an event via JS console *without* logging in returns a 403/Forbidden error from Supabase.

## must_haves
- Public users can still view events without being logged in.
- Authenticated admin can successfully update 'status' to 'APPROVED' or 'REJECTED'.
